## 视觉设计规范

### 色彩体系

**主色调**：黑白灰
- 纯黑 #000000 或深灰 #1a1a1a（主文字）
- 纯白 #FFFFFF 或浅灰 #F8F8F8（背景）
- 中灰 #666666、#999999（辅助文字）

**主题色**：低饱和度冷色调（1-2种）
- 浅蓝：#E3F2FD 或 #ECEFF1（浅色背景）
- 灰蓝：#546E7A 或 #455A64（强调色）
- 使用场景：导航高亮、卡片边框、按钮背景、链接颜色

**辅助色**：
- 成功绿：#4CAF50（表单提交成功）
- 错误红：#F44336（表单错误提示）
- 警告黄：#FFC107（其他提示）

### 字体排版

**无衬线字体**：
- 英文：Inter、-apple-system、Helvetica Neue
- 中文：思源黑体（SourceHanSansCN）、微软雅黑、PingFang SC
- 备选：系统字体栈 `-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto`

**字体大小规范**：
- H1（页面大标题）：48px - 64px
- H2（板块标题）：36px - 48px
- H3（小标题）：24px - 32px
- 正文文字：14px - 16px
- 小字说明：12px - 14px

**行高规范**：
- 标题行高：1.2 - 1.3
- 正文行高：1.5 - 1.8
- 列表行高：1.6 - 1.8

**字间距规范**：
- 标题字间距：0.05em
- 正文字间距：0.02em
- 大段落逐行字数控制在60-80个字为佳

### 间距与布局

**外边距规范**：
- 大板块间间距：80px - 120px
- 小板块间间距：40px - 60px
- 元素间间距：20px - 40px

**内边距规范**：
- 大容器内边距：60px
- 中等容器内边距：40px
- 小元素内边距：20px

**网格系统**：
- 使用8px基准网格
- 所有尺寸都是8px的倍数

### 边框与阴影

**边框**：
- 极简风格，谨慎使用边框
- 使用场景：卡片分隔、表单输入框、按钮
- 边框颜色：浅灰色 #EEEEEE 或 #DCDCDC
- 边框宽度：1px

**阴影**：
- 使用低饱和度阴影，避免深色重阴影
- 标准阴影：`0 2px 8px rgba(0, 0, 0, 0.08)`
- 悬停阴影：`0 8px 24px rgba(0, 0, 0, 0.12)`
- 深阴影（卡片）：`0 4px 16px rgba(0, 0, 0, 0.1)`

### 圆角半径

**圆角规范**：
- 大圆角（按钮、输入框）：8px
- 中等圆角（卡片）：4px
- 小圆角（标签）：4px
- 极简风格：部分元素可不用圆角

## 功能交互规范

### 1. 导航与页面滚动

**导航项点击行为**：
- 点击后页面平滑滚动至对应板块
- 滚动过程触发GSAP轻微过渡动效（可选Parallax视差效果）
- 滚动完成后，导航项自动高亮

**导航高亮逻辑**：
- 通过监听页面滚动位置
- 判断当前可视板块
- 对应导航项添加高亮类名（如 `.active`）
- 过渡动画平滑（使用CSS transition）

**移动端导航**：
- 采用汉堡菜单折叠
- 点击汉堡菜单打开/关闭导航项列表
- 点击导航项后自动关闭菜单并滚动

### 2. 作品分类切换

**标签切换行为**：
- 点击分类标签时，作品卡片平滑切换
- 切换动画：卡片淡出 → 内容更新 → 卡片淡入
- 动画时长：300-400ms
- 切换时保持页面位置不变

**标签样式**：
- 未激活状态：浅灰色文字
- 激活状态：深色文字 + 下划线或背景色
- Hover状态：颜色渐变、光标变化

### 3. 作品卡片交互

**Hover状态效果**：
- 整体轻微放大（缩放 1.02-1.05倍）
- 阴影增强
- 文字颜色变化（标题颜色渐变）
- 底部显示"查看详情"按钮（如初始隐藏）
- 过渡时长：200-300ms

**点击行为**：
- 点击卡片跳转至作品详情页
- 页面跳转时传递作品ID参数（如 `?id=work001`）
- 可选：加载过程显示加载动画

**卡片加载动画**：
- 页面加载时卡片依次渐入
- 使用交错延迟（每张卡片延迟50-100ms）
- 动画效果：渐入 + 微微向上滑动

### 4. 作品详情页交互

**返回按钮**：
- 点击返回作品展示区
- 回到原分类位置
- 可选：平滑滚动至原卡片位置

**大图查看**：
- 点击作品图片打开大图查看
- 支持全屏或模态框展示
- 支持放大/缩小、上下翻页

**图片懒加载**：
- 进入可视区域时加载
- 加载过程显示骨架屏
- 使用Intersection Observer API实现

### 5. 表单交互

**输入框交互**：
- Focus状态：边框颜色变化、阴影增加、背景色变化
- 输入过程：实时验证（可选）
- Blur状态：验证输入内容

**表单验证**：
- 邮箱格式验证：正则表达式验证
- 必填项验证：提交时检查
- 错误提示：在输入框下方显示错误信息
- 成功提示：表单提交后显示成功消息（可选modal）

**提交按钮**：
- 默认状态：可点击
- Hover状态：背景色变化、光标变化
- 禁用状态：灰色、光标不可用（表单无效时）
- 加载状态：显示加载动画、按钮文字变为"提交中..."

### 6. 响应式交互

**桌面端**（≥1200px）：
- 完整导航栏显示

## 动效与GSAP应用

### 动效原则

1. **服务于体验**：所有动效都应服务于用户体验提升，不做过度设计
2. **突出内容**：动效不应抢作品内容焦点，保持克制
3. **流畅性**：动效需流畅自然，无明显卡顿
4. **一致性**：全站动效风格统一

### GSAP应用场景

#### 1. 页面加载动效
```javascript
// Hero区文字依次渐入
gsap.from(".hero-title", {
  duration: 0.8,
  opacity: 0,
  y: 20,
  delay: 0.2
});

// 作品卡片交错渐入
gsap.from(".work-card", {
  duration: 0.6,
  opacity: 0,
  y: 20,
  stagger: 0.1
});
```

#### 2. 平滑滚动动效
```javascript
// 导航项点击平滑滚动
gsap.to(window, {
  duration: 1,
  scrollTo: { y: targetElement, offsetY: 80 },
  ease: "power2.inOut"
});
```

#### 3. 卡片Hover动效
```javascript
// 卡片悬停放大
gsap.to(".work-card", {
  duration: 0.3,
  scale: 1.05,
  boxShadow: "0 8px 24px rgba(0,0,0,0.12)",
  ease: "power2.out"
});
```

#### 4. 分类切换动效
```javascript
// 卡片切换时淡出淡入
gsap.to(".work-grid", {
  duration: 0.3,
  opacity: 0,
  onComplete: () => {
    // 更新内容
    updateWorks();
    gsap.from(".work-grid", {
      duration: 0.3,
      opacity: 0
    });
  }
});
```

#### 5. 时间线动效
```javascript
// 时间线项目依次显示
gsap.from(".timeline-item", {
  duration: 0.6,
  opacity: 0,
  x: -30,
  stagger: 0.15,
  scrollTrigger: {
    trigger: ".timeline",
    start: "top 80%",
    toggleActions: "play none none reverse"
  }
});
```

### 动效配置规范

- **过渡时长**：200-600ms（通常300-400ms）
- **缓动函数**：`power2.inOut`, `power3.out`, `cubic-bezier(0.25, 0.46, 0.45, 0.94)`
- **延迟**：50-200ms（用于交错效果）
- **重复次数**：通常无限制（一次性动效）

## 性能优化

### 1. 图片懒加载

**实现方案**：
- 使用 Intersection Observer API
- 初始加载时用占位符或骨架屏
- 真实图片加载完成后逐步显示

```javascript
// 伪代码
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      loadImage(entry.target);
      observer.unobserve(entry.target);
    }
  });
});

document.querySelectorAll('[data-src]').forEach(img => {
  observer.observe(img);
});
```

### 2. 骨架屏设计

**骨架屏样式**：
- 与整体极简风格统一
- 使用浅灰色背景 #F0F0F0
- 闪烁动画：CSS keyframes实现脉冲效果
- 形状与最终加载内容一致

### 3. 资源优化

**文件大小**：
- 使用WebP格式图片（带PNG降级方案）
- 图片压缩：TinyPNG或类似工具
- CSS/JS代码压缩和打包

**缓存策略**：
- 静态资源添加版本号或hash
- 利用浏览器缓存策略
- CDN分发（可选）

### 4. 代码分割

- 按需加载作品详情页数据
- 使用动态导入（dynamic import）
- 分离CSS（考虑关键CSS内联）
